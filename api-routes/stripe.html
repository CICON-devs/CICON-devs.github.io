Stripe Test Environment

Overview​

The api/stripe folder contains API routes for handling payments, onboarding, and account management using the Stripe API. These routes manage various aspects such as payment intent creation, amending payment intents, creating Stripe accounts, and generating onboarding links for connected accounts. This uses the test environment.

Routes​

1. amend-payment-intent/route.ts​

This route is used to amend the price of a repair job based on the cobbler's assessment. It handles scenarios where the cobbler either agrees or disagrees with the price, or if the repair is not possible. Depending on the cobbler's response, the route captures payments, creates new PaymentIntents, or cancels them.

POST /api/stripe/amend-payment-intent​

  • Purpose: Amends or cancels a PaymentIntent based on cobbler feedback.

  • Input:

    • cobblerId: The Stripe account ID of the cobbler.
    • paymentIntentId: The ID of the existing PaymentIntent.
    • orderId: The ID of the associated order.
    • isRepairable: Boolean indicating if the repair is possible.
    • agreeWithPrice: Boolean indicating if the cobbler agrees with the original price.
    • newPrice: New price (if applicable).
    • reason: Reason for the price amendment or cancellation.

  • Steps:

    • If the repair is possible and the cobbler agrees with the price, the payment is captured.
    • If the repair is possible but the cobbler disagrees with the price, the existing PaymentIntent is canceled and a new one is created with the updated price.
    • If the repair is not possible, the PaymentIntent is updated and canceled.

  • Response:

    • Returns the captured PaymentIntent or the new PaymentIntent's client secret.
    • If the repair is not possible, the PaymentIntent is canceled and the client is informed.

  • Example Response:

    {
      "success": true,
      "paymentIntent": { /* captured PaymentIntent details */ }
    }

2. create-account/route.ts​

This route creates a new connected account in Stripe for cobblers, using the Express account type. The account is created with a default business profile for shoe repair services.

POST /api/stripe/create-account​

  • Purpose: Creates a new Stripe Express account for a cobbler.

  • Steps:

    • Creates a new Stripe account with the country set to "GB" and business type as "individual".
    • Sets the Merchant Category Code (MCC) to "5611" for shoe repair services.

  • Response:

    • Returns the account ID of the newly created Stripe account.

  • Example Response:

    {
      "success": true,
      "accountId": "acct_123456789"
    }

3. create-account-link/route.ts​

This route generates an onboarding link for a Stripe connected account. The generated link directs cobblers to the Stripe onboarding flow, allowing them to complete the necessary account setup.

POST /api/stripe/create-account-link​

  • Purpose: Generates an account onboarding link for a connected Stripe account.

  • Input:

    • accountId: The ID of the connected Stripe account.

  • Steps:

    • Constructs the base URL dynamically from the request headers.
    • Generates an account onboarding link with refresh and return URLs.

  • Response:

    • Returns the account onboarding link URL.

  • Example Response:

    {
      "url": "https://connect.stripe.com/setup/s/abc123"
    }

4. get-payment-intent-details/[paymentIntentId]/route.ts​

This route retrieves the details of an existing PaymentIntent from Stripe. It returns information such as the amount, currency, application fee amount, and net amount.

GET /api/stripe/get-payment-intent-details/[paymentIntentId]​

  • Purpose: Retrieves the details of a PaymentIntent from Stripe.

  • Steps:

    • Retrieves the PaymentIntent using its ID.

  • Response:

    • Returns the full details of the PaymentIntent.

  • Example Response:

    {
      "id": "pi_1Jgq2e2eZvKYlo2C",
      "amount": 5000,
      "currency": "gbp",
      "application_fee_amount": 500,
      "net_amount": 4500,
      // Other PaymentIntent details
    }

5. payment-intent/route.ts​

This route creates a PaymentIntent with a split payment between the platform and a connected cobbler account. The platform takes a 10% fee, and the remaining amount is transferred to the cobbler’s Stripe account.

POST /api/stripe/payment-intent​

  • Purpose: Creates a PaymentIntent with a split payment between the platform and a connected Stripe account.

  • Input:

    • amount: The total amount to be charged (in pence).
    • connectedAccountId: The Stripe account ID of the cobbler.

  • Steps:

    • Splits the payment, with 10% going to the platform and 90% to the cobbler's connected Stripe account.
    • Creates the PaymentIntent with manual capture enabled.

  • Response:

    • Returns the client secret for the created PaymentIntent.

  • Example Response:

    {
      "clientSecret": "pi_1Jgq2e2eZvKYlo2C_secret_WBt6dYH6fhg7mJ"
    }

Conclusion​

These routes facilitate key payment and account management operations within the app, allowing cobblers to manage their Stripe accounts, capture payments, and process split payments. The Stripe API integration is crucial for handling cobbler payouts and managing the overall payment flow within the app.
