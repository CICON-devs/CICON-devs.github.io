Cobbler Stripe Onboarding (Live)

Overview​

The api/stripe-onboarding folder contains API routes for handling the onboarding process of cobblers in a live Stripe environment. These routes allow for creating connected accounts and generating onboarding links, which direct users through the Stripe onboarding process.

Routes​

1. create-account/route.ts​

This route is used to create a new connected Stripe account for cobblers. It operates in the live environment, using the LIVE_STRIPE_SECRET_KEY. The account is created with details like the cobbler’s name, email, phone number (converted to an international format if necessary), and business information.

POST /api/stripe-onboarding/create-account​

  • Purpose: Creates a new connected Stripe Express account for cobblers in the live environment.

  • Input:

    • email: The cobbler's email address.
    • firstName: The cobbler's first name.
    • lastName: The cobbler's last name.
    • phone: The cobbler's phone number (converted to international format if it's a UK number).
    • address: The cobbler's address (including line1, city, postal_code).
    • companyName: The business name.

    Example input:

    {
      "email": "cobbler@example.com",
      "firstName": "John",
      "lastName": "Doe",
      "phone": "07123456789",
      "address": {
        "line1": "123 Shoe Lane",
        "city": "London",
        "postal_code": "EC1A1BB"
      },
      "companyName": "Doe Shoe Repairs"
    }

  • Steps:

    • Converts UK phone numbers starting with 07 to international format (+447).
    • Creates a connected account with Stripe’s Express account type, specifying GB as the country and 5611 as the Merchant Category Code (MCC) for shoe repair services.
    • Sets the cobbler's business details, including their name, phone number, and address.

  • Response:

    • Returns the newly created Stripe account ID.

  • Example Response:

    {
      "success": true,
      "accountId": "acct_live_123456789"
    }

  • Error Responses:

    • 500: Error creating the Stripe account.

2. generate-account-link/route.ts​

This route generates an onboarding link for a Stripe connected account in the live environment. The link directs cobblers to complete the onboarding process, where they can provide additional business details and agree to Stripe’s terms.

POST /api/stripe-onboarding/generate-account-link​

  • Purpose: Generates an account onboarding link for a connected Stripe account in the live environment.

  • Input:

    • accountId: The ID of the connected Stripe account.

    Example input:

    {
      "accountId": "acct_live_123456789"
    }

  • Steps:

    • Constructs the base URL dynamically from the request headers.
    • Generates an account onboarding link with refresh and return URLs, directing the cobbler to complete the onboarding process.

  • Response:

    • Returns the generated account onboarding URL.

  • Example Response:

    {
      "url": "https://connect.stripe.com/setup/s/abc123/live"
    }

  • Error Responses:

    • 500: Error generating the account link.

Conclusion​

The stripe-onboarding routes are essential for managing live onboarding of cobblers in the platform, ensuring that their Stripe Express accounts are correctly created and linked for payout processing. The routes also facilitate smooth onboarding with automatically generated links for cobblers to complete their setup.

This documentation provides a clear overview of how to interact with Stripe in a live environment for the stripe-onboarding folder, including both account creation and link generation.
