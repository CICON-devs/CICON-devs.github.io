Prisma Routes for DB connection

Overview​

The api/record-orders folder contains API routes responsible for managing orders within the shoe repair app. This includes initializing new orders, saving customer details, and saving items related to those orders. The routes interact with a database using Prisma ORM to create, update, and retrieve order-related data.

Routes​

1. initialize-order/route.ts​

This route initializes a new order in the shoe repair app. It takes input such as the prompt and fault information, then stores the order in the database with a default status of "Pending".

POST /api/record-orders/initialize-order​

  • Purpose: Creates a new order in the database.

  • Input:

    • prompt: The text description provided by the customer.
    • fault_information: Detailed fault information provided by the customer.

    Example input:

    {
      "prompt": "My shoes have a tear on the side.",
      "fault_information": {
        "fault_type": "Tear",
        "location": "Side",
        "severity": "Medium"
      }
    }

  • Steps:

    • Accepts the input and converts fault_information into a JSON string.
    • Creates a new order in the orders table with the provided data.
    • Sets the status of the order to "Pending".

  • Response:

    • On success, returns the newly created order along with its orderId.
    • On failure, returns a descriptive error message.

  • Example Response:

    {
      "success": true,
      "order": {
        "id": "order_123",
        "prompt": "My shoes have a tear on the side.",
        "fault_information": "{\"fault_type\":\"Tear\",\"location\":\"Side\",\"severity\":\"Medium\"}",
        "created_at": "2024-09-17T10:00:00Z",
        "updated_at": "2024-09-17T10:00:00Z",
        "status": "Pending"
      },
      "orderId": "order_123"
    }

  • Error Responses:

    • 500: Error initializing order

2. save-customer/route.ts​

This route saves customer details in the database. It also associates the customer with the corresponding order by updating the customer_id in the orders table. If the customer already exists (e.g., due to a unique email), the route will handle this scenario.

POST /api/record-orders/save-customer​

  • Purpose: Saves customer details and associates them with an order.

  • Input:

    • firstName: Customer's first name.
    • lastName: Customer's last name.
    • email: Customer's email address.
    • phone: Customer's phone number.
    • address: Customer's physical address.
    • orderId: The ID of the order to associate the customer with.

    Example input:

    {
      "firstName": "John",
      "lastName": "Doe",
      "email": "john.doe@example.com",
      "phone": "+1234567890",
      "address": "123 Main Street",
      "orderId": "order_123"
    }

  • Steps:

    • Creates a new customer in the customers table.
    • Updates the order with the newly created customer_id.
    • Updates all items in the order with the same customer_id.

  • Response:

    • On success, returns the customer information and their customerId.
    • On failure, returns an appropriate error message (e.g., for duplicate email entries).

  • Example Response:

    {
      "success": true,
      "customer": {
        "id": "customer_456",
        "first_name": "John",
        "last_name": "Doe",
        "email": "john.doe@example.com",
        "phone": "+1234567890",
        "address": "123 Main Street"
      },
      "customerId": "customer_456"
    }

  • Error Responses:

    • 400: Email is already registered
    • 500: Error saving customer details

3. save-items/route.ts​

This route saves the items (both services and products) associated with an order. It takes input such as the order ID, selected services, and selected products, and stores them in the database.

POST /api/record-orders/save-items​

  • Purpose: Saves services and products associated with an order.

  • Input:

    • orderId: The ID of the order these items are linked to.
    • selectedServices: An array of selected service objects.
    • selectedProducts: An array of selected product objects.

    Example input:

    {
      "orderId": "order_123",
      "selectedServices": [
        { "name": "Heel Repair", "description": "Repair the heel", "price": 20 }
      ],
      "selectedProducts": [
        { "name": "Shoe Polish", "description": "Polish for leather shoes", "price": 10 }
      ]
    }

  • Steps:

    • Ensures that an order ID and at least one item (service or product) are provided.
    • Prepares the items and saves them in the products_and_services_items table.

  • Response:

    • On success, returns a confirmation that the items were saved.
    • On failure, returns an appropriate error message.

  • Example Response:

    {
      "success": true,
      "items": {
        "count": 2
      }
    }

  • Error Responses:

    • 400: Missing order ID or no valid items to save
    • 500: Error saving services and products

Conclusion​

The record-orders routes provide a streamlined process for managing orders, customers, and their associated items (services and products) within the shoe repair app. These routes allow for order initialization, customer association, and the saving of repair services and products.
