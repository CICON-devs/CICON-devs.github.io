Cobbler Prices Routes

Overview​

The api/cobbler folder contains API routes responsible for handling cobbler-related functionality, particularly around fetching and saving prices for cobbler services. These routes interact with Excel files to store and retrieve pricing data, which can be amended by the cobbler as needed.

Routes​

1. get-prices/route.ts​

This route is responsible for reading an Excel file that contains cobbler pricing data and returning it as JSON. The Excel file includes details such as the type of shoes, materials used, faults, and the price for repairs.

GET /api/cobbler/get-prices​

  • Purpose: Fetches cobbler pricing details from an Excel file and returns the data as JSON.

  • Excel Columns:

    1. Shoes
    2. Material
    3. Part of Item
    4. Fault
    5. Fix
    6. Size
    7. Price
    8. Price Pair

  • Steps:

    • Reads an Excel file from the public directory (prices.xlsx).
    • Logs available sheet names for debugging purposes.
    • Converts the first worksheet into a JSON object, skipping the header row.

  • Response:

    • If successful, returns the prices as JSON.
    • If an error occurs, returns a descriptive error message with an appropriate HTTP status code.

  • Example Response:

    {
      "prices": [
        {
          "shoes": "Boots",
          "material": "Leather",
          "part_of_item": "Sole",
          "fault": "Tear",
          "fix": "Stitching",
          "size": "Medium",
          "price": 50,
          "price_pair": 90
        },
        ...
      ]
    }

  • Error Responses:

    • 404: Excel file not found
    • 500: Worksheet not found or other server errors

2. save-prices/route.ts​

This route allows cobblers to save or update their pricing data. The data is sent from the client as JSON, converted into an Excel format, and saved to a new file.

POST /api/cobbler/save-prices​

  • Purpose: Saves cobbler pricing data to a new Excel file.

  • Input: The request body contains an array of objects, each representing a pricing record:

    {
      "prices": [
        {
          "shoes": "Boots",
          "material": "Leather",
          "part_of_item": "Sole",
          "fault": "Tear",
          "fix": "Stitching",
          "size": "Medium",
          "price": 50,
          "price_pair": 90
        }
      ]
    }

  • Steps:

    • Receives the prices from the request body.
    • Creates a new Excel workbook and adds a worksheet titled "Prices".
    • Adds the appropriate headers: Shoes, Material, Part of Item, Fault, Fix, Size, Price, Price for Pair.
    • Iterates over the pricing data and populates the worksheet with rows.
    • Saves the updated Excel file to a new directory (./new_directory/Updated_Prices.xlsx).

  • Response:

    • If successful, returns a success message.
    • If an error occurs, returns an error message with an appropriate HTTP status code.

  • Example Response:

    {
      "success": true,
      "message": "Prices saved successfully!"
    }

  • Error Responses:

    • 500: Error saving prices

Conclusion​

These two routes provide the essential functionality for cobblers to fetch and update their service pricing, ensuring the data is securely stored in Excel format and easily accessible through the API.
