Main Repair Flow

The main repair flow of the application is a multi-step process that guides users through submitting their shoe repair details, selecting repair services, and completing the checkout process. Each stage of the process is rendered dynamically based on the URL stage parameter.

Stages Overview​

Stages:​

  1. Prompt and Photo Upload Stage: Users provide details about their shoes and upload photos.
  2. Analysis Stage: The application analyzes the uploaded photos and generates repair suggestions.
  3. Repair Selection Stage: Users select repair services and additional product recommendations for their shoes.
  4. Checkout Stage: Users review their order and complete the payment process.
  5. Payment Success Stage: The final stage where users are shown a summary of their order and payment status.

Stage Flow (app/stages/[stage]/page.tsx)​

This component renders different child components based on the current stage, such as uploading images, analyzing faults, selecting repairs, completing the order, and making payments.

Key Hooks Used:​

  • useParams: Fetches the stage parameter from the URL.
  • useRouter: Manages navigation between stages.
  • useStageContext: Accesses shared state for each stage.

Functions:​

  • handleBasketToDetails: Transitions from repair selection to order details by setting the total amount.
  • handleDetailsToCheckout: Proceeds from order details to the checkout stage.
  • renderStageComponent: Renders the appropriate stage based on the current URL parameter.

Prompt and Photo Upload Stage (app/stages/PromptAndPhotoUploadComponent.tsx)​

In this stage, users:

  • Provide prompts describing their shoe repairs.
  • Upload up to 3 photos per repair.

Features:​

  • Multiple repair sections can be added, up to a limit of 3.
  • Each section has a prompt and file upload area.
  • Files are validated (e.g., max 3 files per repair section).
  • User is required to input prompts and upload photos before proceeding to the next stage.

Analysis Stage (app/stages/AnalysisComponent.tsx)​

This stage processes the uploaded photos and prompts to generate repair suggestions. The application:

  • Sends the images and prompts to a backend service for analysis.
  • Receives fault analysis, repair services, and product suggestions.
  • Automatically transitions to the repair selection stage once the analysis is complete.

Key Points:​

  • If faults or files are missing, an error message is displayed.
  • The analysis result (fault info) is stored in the context for the next stage.

Repair Selection and Order Completion Stage (app/stages/ProductsAndServicesComponent.tsx)​

In this stage, users:

  • View suggested repair services based on the analysis results.
  • Select between repair options (e.g., "pair" or "single" shoe repair).
  • Optionally add recommended products to their order.

Features:​

  • Services and products are dynamically displayed based on analysis.
  • Users can toggle between repair options and add items to the cart.
  • A cart summary displays selected services, products, and the total amount.

Order Details Stage (app/stages/DetailsComponent.tsx)​

Users enter their personal and shipping details, such as:

  • First Name
  • Last Name
  • Email Address
  • Phone Number
  • Street Address, City, and Postal Code

Key Points:​

  • The entered details are stored in the context and linked to the current order.
  • Once submitted, the application proceeds to the checkout stage.

Checkout Stage (app/stages/CheckoutComponent.tsx)​

In this stage, users:

  • Review their selected services, products, and the total amount.
  • Complete the payment process using Stripe’s CardElement.

Features:​

  • Payment processing with Stripe’s API.
  • Payment is held until the cobbler confirms the repair job.
  • Errors during payment are handled gracefully.

Payment Success Stage (app/components/PaymentReviewComponent.tsx)​

After a successful payment, users are shown:

  • A summary of the services and products they’ve selected.
  • The total amount paid.
  • A message informing them that the payment is on hold until the cobbler confirms the order.

Flow Summary:​

  1. User uploads shoe details and photos.
  2. System analyzes faults and generates service and product suggestions.
  3. User selects repair services and optional products.
  4. User enters personal details and confirms the order.
  5. User reviews and completes the payment.
  6. System confirms the payment and notifies the user about the order status.

This flow handles both the user interaction and backend communication to process orders, analyze faults, and facilitate payments using Stripe.
